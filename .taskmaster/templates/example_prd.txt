# 개요 (Overview)

**집체크(zipcheck.com)**  
전세사기 피해 예방 및 계약 이후 준비사항 가이드를 제공하는 서비스.  
사용자가 등기부등본을 업로드하면 AI 기반으로 분석해 위험도를 보여주고, 계약 후 준비해야 할 절차를 안내하여 임차인의 불안을 해소합니다.  
주 타겟은 전세 계약 예정자 또는 계약 후 불안감을 느끼는 임차인입니다.

# 핵심 기능 (Core Features)

### 1. 등기부등본 기반 자동 분석

- **무엇:** 등기부등본 PDF 업로드 → Python API로 텍스트 추출 → OpenAI GPT로 전세사기 체크리스트 분석
- **왜 중요한가:** 사전에 문제를 발견해 피해를 예방할 수 있음
- **어떻게:** 사용자가 PDF 업로드 시 자동으로 분석하여 위험도 및 체크리스트 형태로 결과 제공

### 2. 주소 및 분석 기록 관리

- **무엇:** 사용자가 분석한 주소와 결과를 저장
- **왜 중요한가:** 추후 재확인 및 비교 가능
- **어떻게:** 업데이트 일자 기록 후 WatchList 형태로 관리 (단순 기록용)

### 3. 계약 이후 준비사항 가이드

- **무엇:** 계약 정보 기반으로 필요한 후속 절차 안내
- **왜 중요한가:** 계약 후 보증금 보호를 위한 실질적인 대응 가능
- **어떻게:** 확정일자 신고, 전입신고, 보증보험 가능 여부 등을 사용자 맞춤형으로 제공

### 4. 등기부등본 발급 안내

- **무엇:** 등기부등본 발급 방법 설명 페이지 제공
- **왜 중요한가:** 서비스 사용에 필수인 문서 발급을 돕기 위해
- **어떻게:** 단계별 발급 가이드 제공

# 사용자 경험 (User Experience)

### 사용자 페르소나

1. 30대 초반 직장인, 첫 전세 계약 예정, 부동산 지식 부족
2. 계약 후 불안감을 느끼는 임차인 → 준비사항 가이드 기대

### 주요 사용자 흐름 (User Flow)

1. 로그인
2. 서비스 설명 → 시작하기
3. 등기부등본 파일 업로드
4. 분석 결과 확인
5. 계약 정보 입력
6. 준비사항 가이드 확인
7. 분석 내역(업로드 기록) 확인

### UI/UX 고려사항

- Shadcn UI 사용 (단색 테마, 화이트/블랙 전환 가능)
- 모바일/웹 하이브리드 대응
- 도메인 구분:
  - 모바일: `m.zipcheck.com`
  - 웹: `zipcheck.com`
- 메뉴 단순화:
  - 분석내역
  - 내정보

# 기술 아키텍처 (Technical Architecture)

- **시스템 구성 요소:**

  - 프론트엔드: React 18 + TailwindCSS + Shadcn UI
  - 백엔드: Supabase Edge Functions (Deno)
  - PDF 텍스트 추출 API: Python (FastAPI)
  - GPT 분석: OpenAI GPT API
  - DB: Supabase PostgreSQL

- **데이터 모델:**

  - 사용자 정보
  - 분석된 주소 및 등기부등본 정보, 결과
  - 업데이트 일자 기록
  - WatchList (단순 기록용)

- **API 및 외부 서비스 연동:**

  - PDF 텍스트 추출 (Python API)
  - OpenAI GPT API
  - (향후) 국토부 API/등기통계 API 연동

- **인프라 요구사항:**
  - 프론트 배포: Vercel 기반
  - 서버리스 구조 (Supabase Edge Functions)
  - 비용 모니터링 및 최적화 필요

- UI/UX 고도화

### 확장 단계

# 기능 개발 순서 (Logical Dependency Chain)

1. 사용자 인증 (Supabase Auth)
2. 등기부등본 업로드 기능
3. Python 텍스트 추출 API + GPT 분석
4. 분석 결과 화면 구성
5. 계약 정보 입력 및 준비사항 가이드 제공
6. 분석 기록 저장 기능 (분석내역)
7. UI/UX 개선
8. 외부 API 연동 고도화
9. 리포트 다운로드 및 B2B 확장

# 리스크 및 대응 방안 (Risks and Mitigations)

- **PDF 품질 저하:** OCR fallback 설계
- **GPT 분석 정확도:** Prompt 고도화 및 테스트 케이스 확보
- **서버비용 증가:** 서버리스 구조 활용, 비용 모니터링
- **대용량 데이터:** 인덱싱 및 파티셔닝 설계 필요
- **1인 개발 리스크:** 문서화, 자동화 테스트, 커뮤니티 피드백 적극 반영
